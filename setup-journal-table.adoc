= Set up the Journal table infrastructure for NetApp Workload Factory
:icons: font
:imagesdir: ./media/

[.lead]
Set up the Journal table infrastructure to generate logs on access points in a table to audit user access events and operations on objects in your AWS S3 buckets. The Journal table includes CloudTrail events, trails (a record of AWS activities), CloudTrail filters, CloudWatch log groups, and more. Several steps are necessary to set up the infrastructure for AWS CloudTrail, AWS CloudWatch, AWS S3 Buckets, AWS CloudWatch log group, and AWS Identity and Access Management (IAM), ObservabilityAdmin, and AWS S3 Tables to enable the journal table feature. This includes creating a CloudTrail trail to log events, setting up CloudWatch log groups to store logs, and configuring IAM roles and policies to allow Workload Factory to access the necessary resources.

== About this task
The Journal feature captures S3 data-plane events (PutObject, GetObject, DeleteObject, etc.) for monitored FSx S3 access points. It uses a chain of AWS services deployed into the customer's account: 

[cols="1,1,1",options="header"]
|===
|AWS service |Resource name pattern | Purpose

|AWS CloudFormation
|netapp-metadata-* 
|Deploys all infrastructure as a stack 

|AWS S3 bucket
|netapp-metadata-cloudtrail-events-logs-{uuid} 
|Stores raw CloudTrail log files 

|AWS CloudTrail
|netapp-metadata-journal-data-events-trail-{uuid} 
|Captures S3 data events for specific access points 

|AWS CloudWatch log group
|netapp-metadata-journal-data-events-{uuid} 
|Receives CloudTrail events as structured log entries 

|IAM roles
a|
* netapp-metadata-cloudtrail-cw-role-{uuid} 
* netapp-metadata-s3table-integration-role-{uuid} 
|Active integration

|ObservabilityAdmin 
|S3TableIntegration 
|Bridges CloudWatch Logs into an S3 Tables table 

|S3 Tables
|aws-cloudwatch bucket → logs.aws_cloudtrail__data 
|Stores structured, queryable CloudTrail events (Iceberg format) 
|===

The {uuid} is a random 8-character identifier generated when the template is created. 

== Before you begin
To enable the journal table feature, you must do the following: 

* You must have an existing volume with an S3 access point.
* Set network configuration to *Internet* for the S3 access point. link:edit-s3-access-point.adoc[Edit network configuration for the S3 access point].
* You must link:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html[grant credentials with the _operations and remediation_ permission policy^] in Workload Factory to complete this task and add the following IAM policy permissions to your AWS credentials used for Workload Factory.

== Set up the journal table infrastructure 
Set up the architecture to connect infrastructure to the S3 access point and create a path for events from AWS services to be captured in the journal table. 

.Steps
. Log in using one of the link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html[console experiences^].
. Select the menu image:workloads-menu-icon.png[The hamburger menu icon is used to navigate to workloads like storage, eda, ai, databases, vmware, and administration.] and then select *Storage*.
. From the Storage menu, select *FSx for ONTAP*.
. From *FSx for ONTAP*, select the actions menu of the file system with the volume to update, then select *Manage*.
. From the file system overview, select the *Volumes* tab.
. From the Volumes tab, select the actions menu for the volume you want to manage S3 access points for, then select *Advanced actions*, and then *Manage S3 access points*.
. From the *Manage S3 access points* screen, select the actions menu and then select *Edit access point*.
. In the *Edit S3 access point* dialog, ensure that the network configuration is set to *Internet*.
. Follow the instructions in the dialog to set up the infrastructure for the journal table feature.
. Download the CloudFormation template.
. Deploy the CloudFormation stack in your AWS account.
.. Save the template JSON to a file.
.. Deploy the template using the AWS CLI or AWS Management Console.
.. Wait for the stack to reach `CREATE_COMPLETE` status.
.. Get the CloudTrail ARN from the stack outputs.
. Return to the Workload Factory console and back to the volume to manage S3 access points for. 
. Select *View details* from the volume actions menu. 

If any of the steps fail, <<troubleshoot the infrastructure setup,Troubleshoot the infrastructure setup for the journal table>> or contact NetApp support for assistance.
